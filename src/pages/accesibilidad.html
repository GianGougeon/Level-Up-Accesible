Accesibilidad

Comprometidos con una experiencia inclusiva para todos

Nuestro compromiso con la accesibilidad

En Level-Up Accesible no solo hablamos de gaming inclusivo, lo practicamos en nuestro propio sitio web. Hemos implementado las pautas WCAG 2.1 AA para garantizar que todos los usuarios, independientemente de sus capacidades, puedan navegar y disfrutar de nuestro contenido.

Este sitio ha sido diseñado y desarrollado siguiendo las mejores prácticas de accesibilidad web, con pruebas realizadas por usuarios con diversas discapacidades y con tecnologías asistivas.

Principales características de accesibilidad

Estructura semántica

Uso adecuado de encabezados (h1-h6), landmarks y roles ARIA para una navegación lógica.

WCAG 1.3.1 WCAG 2.4.10

Navegación consistente

Menú de navegación repetido en todas las páginas con orden lógico y accesible por teclado.

WCAG 3.2.3

Enlaces descriptivos

Textos de enlace significativos fuera de contexto, evitando "clic aquí" o "leer más".

WCAG 2.4.4

Migas de pan

Ruta de navegación visible para orientar al usuario sobre su ubicación en el sitio.

WCAG 2.4.8

Texto legible

Contenido claro y simple, con opción de aumentar el tamaño del texto sin pérdida de funcionalidad.

WCAG 1.4.4

Texto alternativo

Imágenes con descripciones precisas, decorativas marcadas como tales para lectores de pantalla.

WCAG 1.1.1

Multimedia accesible

Videos con subtítulos, transcripciones y descripción de audio cuando es necesario.

WCAG 1.2.2 WCAG 1.2.3

Tablas accesibles

Tablas con encabezados identificados correctamente para una interpretación adecuada.

WCAG 1.3.1

Contraste adecuado

Relación de contraste mínimo de 4.5:1 para texto normal y 3:1 para texto grande.

WCAG 1.4.3

Independencia del color

La información no se transmite solo mediante color, usando también texto o patrones.

WCAG 1.4.1

Redimensión de texto

El texto puede ser ampliado hasta un 200% sin pérdida de contenido o funcionalidad.

WCAG 1.4.4

Modo de alto contraste

Opción para activar un esquema de colores de alto contraste para mejor legibilidad.

WCAG 1.4.6

Navegación por teclado

Toda la funcionalidad está disponible usando solo el teclado, sin trampas de teclado.

WCAG 2.1.1

Foco visible

Indicador claro del elemento con foco durante la navegación por teclado.

WCAG 2.4.7

Saltar al contenido

Enlace oculto visible al navegar con teclado para saltar bloques repetitivos.

WCAG 2.4.1

Manejo de errores

Formularios con validación accesible y sugerencias para corregir errores.

WCAG 3.3.1 WCAG 3.3.3

Detalle de implementación WCAG 2.1

Principio Pauta WCAG Nivel Implementación en este sitio

1.4.1 Uso del color A La información no se transmite solo con color (ej. enlaces subrayados)

1.4.4 Redimensión del texto AA Texto redimensionable hasta 200% sin pérdida de funcionalidad

2.4.3 Orden del foco A Orden lógico de tabulación que preserva las relaciones significativas
2.4.4 Propósito de los enlaces (en contexto) A Textos de enlace descriptivos fuera de contexto
2.4.7 Foco visible AA Indicador de foco claro con alto contraste en todos los elementos interactivos

3.2.3 Navegación consistente AA Estructura de navegación consistente en todas las páginas

Robusto 4.1.1 Procesamiento A HTML válido, sin errores de sintaxis que afecten la interpretación
4.1.2 Nombre, función, valor A Componentes con nombres accesibles, estados y propiedades definidos

Tecnologías asistivas compatibles

Lectores de pantalla

Probado con:

  • NVDA (Windows)
  • VoiceOver (Mac/iOS)
  • TalkBack (Android)
  • JAWS (Windows)

Navegación por teclado

Compatible con:

  • Teclado físico
  • Teclado en pantalla
  • Conmutadores
  • Dispositivos de entrada alternativos

Herramientas de adaptación

Optimizado para:

  • Lupas de pantalla
  • Modo alto contraste
  • Software de reconocimiento de voz
  • Herramientas de dictado
